# Claude Code Instructions

- Be terse and concise in all responses
- Focus on ONE task at a time - complete it fully before moving to the next
- Only fix one bug at a time
- Only implement one function at a time
- Only change one line at a time, just one line (figuratively speaking)
- Minimize explanations unless specifically asked
- Avoid suggesting additional improvements or related changes unless requested
- Include explanations when proposiing changes to files
- Read and understand any .md files in the project to make sure you are following the project requirements
- Always write tests for new or exisitng functionality
- Always propose modular solutions
- Always avoid adding code to existing files unless they are directly related (dependencies)
